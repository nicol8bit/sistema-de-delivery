# Sistema de Delivery

**Requisitos Funcionais:**

Cliente: 

1. O sistema deve permitir que o Cliente se cadastre e faça login. E deve permitir a gestão de seu perfil e endereços de entrega.   
2. O sistema deve permitir que o Cliente busque restaurantes e produtos por nome, categoria, localização e aplique filtros para pesquisar.  
3. O sistema deve permitir que o Cliente visualize o menu completo de um restaurante, incluindo fotos, descrições, preços e opções de customização.  
4. O sistema deve permitir que o Cliente adicione, remova e modifique itens no carrinho de compras.  
5. O sistema deve permitir que o Cliente finalize o pedido, selecionando endereço e forma de pagamento.  
6. O sistema deve permitir que o Cliente acompanhe o status do pedido em tempo real.  
7. O sistema deve permitir que o Cliente avalie o restaurante e o entregador após a conclusão do pedido.  
8. O sistema deve permitir que o Cliente visualize seu histórico de pedidos.

Restaurante: 

1. O sistema deve permitir que o Restaurante se cadastre e faça login. Deve permitir a gestão de seu perfil, horário de funcionamento e área de entrega.  
2. O sistema deve permitir que o Restaurante cadastre, edite e remova categorias, produtos e customize o cardápio.  
3. O sistema deve notificar o Restaurante sobre novos pedidos e permitir que ele aceite ou rejeite o pedido.  
4. O sistema deve permitir que o Restaurante atualize o status do pedido.

Entregador: 

1. O sistema deve permitir que o Entregador se cadastre e faça login. Deve permitir a gestão de seu perfil e status de disponibilidade.  
2. O sistema deve notificar o Entregador sobre novas solicitações de entrega e exibir detalhes.  
3. O sistema deve permitir que o Entregador aceite ou rejeite uma solicitação de entrega.  
4. O sistema deve fornecer a rota para retirada e entrega, relacionando-se a serviços de mapas.  
5. O sistema deve permitir que o Entregador atualize o status da entrega.

**Requisitos Não Funcionais:** 

1. Todas as comunicações entre o cliente, restaurante, entregador e o servidor devem ser criptografadas.  
2. A interface do usuário deve ser intuitiva.  
3. O aplicativo do Cliente e do Entregador deve ser compatível com os sistemas operacionais Android e iOS.  
4. O sistema deve ser capaz de suportar um grande aumento no número de usuários e pedidos simultâneos sem degradação perceptível de desempenho.  
5. O processamento de pagamentos deve estar em conformidade para proteção de dados de cartão.
